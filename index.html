Flexbox: uma poderosa ferramenta na construção de layouts

Por Marcelo Paschoal, 16 de maio de 2025.

Flexbox

O Flexbox é um modelo de layout em CSS que facilita a criação de interfaces mais flexíveis e adaptáveis. Ele permite organizar elementos de forma unidimensional — em linha ou em coluna — dentro de um flex container, ou seja, um contêiner que utiliza o Flexbox para controlar diretamente a distribuição de seus elementos filhos. Com ele, é possível definir com facilidade o alinhamento, o tamanho, o espaçamento e a ordem dos itens, o que contribui para layouts mais limpos e eficientes em diferentes tamanhos de tela. O Flexbox também veio para simplificar práticas antigas de criação de layout baseadas em propriedades como float e position, oferecendo uma solução mais intuitiva e poderosa. Para utilizá-lo, basta aplicar display: flex ao elemento contêiner.

Conceitos fundamentais

Mas antes de continuarmos, precisamos definir alguns conceitos importantes que nos auxiliarão no decorrer deste artigo. Entender como funcionam o main axis, cross axis, main size, cross size, main start, main end, cross start e cross end é fundamental para aplicar corretamente as principais propriedades do Flexbox. A seguir, apresentamos a explicação de cada um deles.

  • Eixos (Axis)

    O main axis é o eixo principal no qual os elementos filhos do contêiner flex são dispostos, variando de acordo com a direção definida — em linha ou em coluna. Ele não está relacionado diretamente às orientações horizontal ou vertical, mas sim à propriedade flex-direction, que abordaremos mais à frente. O cross axis, por outro lado, é o eixo perpendicular ao main axis, sendo fundamental para compreender o alinhamento dos elementos filhos por meio da propriedade align-items.

  • Tamanho (Size)

    O main size é o tamanho total do contêiner flex em relação ao main axis, enquanto o cross size é o tamanho total do contêiner flex em relação ao cross axis.

  • Início e fim do contêiner (Start and End)

    Tanto o main start quanto o cross start definem o início de um contêiner em relação ao eixo principal ou ao eixo perpendicular. Por outro lado, o main end e o cross end delimitam o fim do contêiner em relação a esses eixos.

Flex direction

A propriedade flex-direction é responsável por definir a direção dos elementos filhos dentro do container flex. Ao alternar seus valores, é possível organizar os elementos em linha, com o é possível dispor os elementos em coluna com column, sendo exibidos de cima para baixo, ou de forma invertida column-reverse.

Justify content

A propriedade justify-content é responsável por alinhar o conteúdo de um container ao longo de seu main axis, definindo como os elementos serão distribuídos nesse eixo. Ao utilizarmos essa propriedade, temos algumas opções bem interessantes. Com o valor flex-start, os itens são agrupados no início do main axis (também chamado de main start). Já com flex-end, temos o comportamento oposto, com os elementos agrupados no final (main end). Quando usamos center, os itens são centralizados ao longo do eixo principal.

Mas essas não são as únicas opções. Também é possível definir espaçamentos entre os elementos do container. Com space-between, todos os itens são separados por espaços iguais, sendo que o primeiro fica encostado no main start e o último no main end. Com space-around, todos os itens recebem o mesmo espaçamento dos dois lados, o que resulta em um espaço duplo entre os elementos internos e um espaço simples nas extremidades. Por fim, com space-evenly, garantimos que os elementos tenham exatamente os mesmos espaços entre si e também entre as bordas do container.

Experimente:

justify-content: flex-start

Align items

Com a propriedade align-items, conseguimos definir como os elementos filhos de um container flex serão alinhados ao longo do cross axis (eixo perpendicular ao main axis). Por padrão, um container flex já possui essa propriedade definida como stretch, fazendo com que todos os elementos filhos ocupem automaticamente todo o espaço disponível no cross axis, esticando-se para igualar a altura (ou largura, direção do flex).

Esse comportamento pode ser ajustado com outros valores, dependendo da necessidade do layout. Com center, os itens são centralizados ao longo do cross axis. Se quisermos que os itens fiquem alinhados ao início ou ao fim desse eixo, usamos flex-start ou flex-end, respectivamente. Também é possível alinhar os elementos com base na linha de base de seu conteúdo textual utilizando o valor baseline. Ao aplicar qualquer um desses valores, o comportamento stretch é desativado, e os itens passam a ocupar apenas o espaço necessário no cross axis.

Experimente:

align-items: stretch

Align content

A propriedade align-content, por sua vez, não organiza os elementos individualmente em relação ao cross axis, mas sim as linhas do container flex. O valor padrão é stretch, que funciona de maneira semelhante ao align-items, esticando as linhas para ocupar todo o espaço disponível no container. No entanto, essa propriedade só tem efeito quando o container possui múltiplas linhas, o que exige a aplicação da propriedade flex-wrap que será abordada mais a frente.

Os valores flex-start, flex-end e center também funcionam de forma parecida com o align-items, mas aplicam o alinhamento às linhas do container, e não diretamente aos seus elementos. Além disso, dois novos valores são específicos dessa propriedade: space-between, que distribui as linhas igualmente, deixando a primeira encostada no cross start e a última no cross end; e space-around, que distribui as linhas com espaços iguais ao redor de cada uma, criando margens entre todas as linhas e também entre as extremidades do container.

Experimente:

align-content: stretch

Flex wrap

Caso o conjunto de elementos dentro de um container flex seja muito grande, podemos enfrentar o problema de "estouro" dos limites do container. Quando isso ocorre, podemos usar a propriedade flex-wrap, que permite a quebra de linha entre os elementos do container.

O valor padrão é nowrap, ou seja, todos os elementos permanecem em uma única linha, mesmo que ultrapassem os limites do container. Podemos alterar esse comportamento com o valor wrap, que permite a quebra de linha e faz com que os itens fluam para linhas subsequentes. Há também o valor wrap-reverse, que aplica a quebra de linha invertendo a direção em que as novas linhas são adicionadas.

Experimente:

flex-wrap: nowrap

Há também outra maneira de aplicar a quebra de linhas ao mesmo tempo em que definimos a direção do container: por meio da propriedade flex-flow. Essa é uma shorthand que combina as propriedades flex-direction e flex-wrap em uma única linha.

Alguns dos valores possíveis incluem: row nowrap, que define a direção principal como horizontal e impede a quebra de linha; row wrap, que mantém a direção horizontal, mas permite a quebra de linha; column nowrap, que organiza os itens verticalmente sem quebra; e column wrap, que organiza os itens em coluna e permite a quebra para novas colunas quando necessário.

Elementos filhos

Podemos criar layouts muito mais interessantes se aplicarmos os conceitos do Flexbox não apenas no container, mas também em seus elementos filhos. Com algumas propriedades como align-self, order, flex-grow, flex-shrink e order, podemos alterar a forma como esses elementos reagem em relação ao container, criando visuais únicos que podem fazer total diferença no design do seu site.

Align self

Sabemos que alguns filhos gostam de ser rebeldes e ir na contramão do que está sendo proposto pelos seus “pais” — e com o Flexbox não é diferente! Quando definimos o alinhamento com align-items, estamos estabelecendo a posição padrão dos itens em relação ao container flex. No entanto, cada elemento pode ajustar seu próprio posicionamento individual utilizando a propriedade align-self.

Entre os valores disponíveis para align-self, temos auto, que determina automaticamente a posição do elemento com base na configuração do container. Além disso, é possível usar os mesmos valores apresentados anteriormente na seção de align-items, como flex-start, flex-end, center e stretch.

Experimente:

align-self: auto

Order

Nem mesmo a ordem dos elementos em um container flex precisa ser a mesma declarada no arquivo HTML! Com a propriedade order, podemos definir, elemento por elemento, qual deles aparecerá primeiro.

A propriedade order recebe valores numéricos e, quanto menor o valor, maior a prioridade na exibição. Ou seja, elementos com valores menores serão exibidos antes dos que tiverem valores maiores. O valor padrão é 0, mas você pode usar números negativos ou positivos para reorganizar os itens conforme desejar.

order: 0

Flex grow

Com a propriedade flex-grow, você pode definir qual proporção do espaço disponível no container flex cada elemento irá ocupar. Essa propriedade é especialmente útil quando há espaço sobrando e você quer distribuí-lo de forma controlada entre os itens.

O valor padrão é 0, o que significa que o item não crescerá além do seu tamanho base. Ao atribuir valores maiores, você determina o quanto aquele item pode crescer em relação aos demais. Por exemplo, se um elemento tem flex-grow: 2 e outro flex-grow: 1, o primeiro ocupará o dobro do espaço disponível em relação ao segundo.

Experimente:

flex-grow: 1

1

Flex shrink

A propriedade flex-shrink define se um elemento filho pode encolher quando o espaço no container flex for insuficiente. Por padrão, seu valor é 1, o que significa que os elementos podem diminuir de tamanho para se ajustarem ao espaço disponível.

Para evitar que um item encolha, podemos definir flex-shrink como 0. Assim, ele manterá seu tamanho base, mesmo que isso provoque o "estouro" do layout ou o aparecimento de rolagem.

Experimente:

flex-shrink: 1

Flex basis

Ao utilizar a propriedade flex-basis, estamos determinando qual será o tamanho base do elemento dentro do container flex. Esse valor define quanto espaço o item ocupará inicialmente, antes de qualquer ajuste feito pelas propriedades flex-grow ou flex-shrink.

Ou seja, mesmo que o espaço entre os elementos seja redistribuído, o elemento com flex-basis definido terá como ponto de partida o valor informado. O valor padrão é auto, que geralmente corresponde ao tamanho do conteúdo interno ou ao valor de width.

Experimente:

flex-basis: 25%

Flex

A propriedade flex é uma shorthand que resume o uso de três propriedades do Flexbox em uma única linha: flex-grow, flex-shrink e flex-basis, nessa ordem. Utilizar essa forma abreviada costuma ser mais prática e clara.
